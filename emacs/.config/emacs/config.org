* How to read this book
:PROPERTIES:
:CUSTOM_ID: intro
:END:

This file *is* your Emacs configuration. Every ~emacs-lisp~ block below is
extracted ("tangled") into =config.el= and loaded at startup. The prose
between blocks is the manual for your own editor. Change a block, save,
and the file re-tangles itself — restart Emacs (or eval the block in
place) and the change is live.

Three keys give you the whole reading experience:

- =TAB= on a heading :: fold / unfold it. =S-TAB= cycles the whole file.
- =C-c C-c= in a code block :: execute it *right now*, in the running editor.
- =C-c C-o= on a link :: follow it — including links that open manuals or
  run code.

And the single most important key in all of Emacs: =C-g= — *cancel*.
Whatever is happening, =C-g= gets you out. (In evil-land =ESC= mostly does
the same job; =C-g= works even where evil isn't.)

Try it immediately. Put your cursor in this block and press =C-c C-c=
(notation: =C-c= means Control+c — so Ctrl-c, Ctrl-c):

#+begin_src emacs-lisp :tangle no :results silent
(message "Hello %s. This editor is now partially made of you."
         user-login-name)
#+end_src

Look at the bottom line of the screen (the /echo area/) — your message is
there. You just evaluated Lisp inside your editor. That feedback loop is
the entire reason Emacs exists.

** Your first-run checklist [0/6]

These need a one-time action each. Mark them done with =C-c C-c= on the
checkbox as you go — the =[0/6]= cookie above updates itself.

- [ ] Let the first launch finish: ~30 packages install and
  native-compile. Watch the =*Messages*= buffer scroll by. One-time cost.
- [ ] Run [[elisp:(nerd-icons-install-fonts t)][M-x nerd-icons-install-fonts]] — installs the icon font the
  modeline and completion menus use. Then restart Emacs.
- [ ] Run [[elisp:my/treesit-install-all][M-x my/treesit-install-all]] — compiles tree-sitter grammars
  (needs =gcc=, which you have).
- [ ] Install language servers (same servers your nvim used, but from
  pacman instead of mason): ~sudo pacman -S --needed clang pyright
  rust-analyzer typescript-language-server lua-language-server~
- [ ] Terminal chapter (ghostel): nothing to install — the native
  module is a prebuilt binary that auto-downloads on first ~M-x ghostel~.
- [ ] Press =SPC h k= and then press any key you're curious about. Emacs
  will tell you exactly what that key runs and link to its source. This
  is /self-documenting software/ — your most powerful learning tool here.

If anything ever goes irreparably wrong: ~emacs --init-directory=/tmp/x~
starts a pristine Emacs that ignores this config entirely. And if the
first launch was interrupted mid-install and packages are half-broken,
the cure is ~rm -rf ~/.config/emacs/elpa~ — next launch reinstalls
everything cleanly.

* Act I — Ignition
:PROPERTIES:
:CUSTOM_ID: early-init
:END:

Emacs reads =early-init.el= before it builds the first frame or touches
packages. Two things belong there and nowhere else.

First: garbage collection. The default GC threshold (800kB) is from the
1980s; during startup it triggers hundreds of collections. We raise it to
infinity for startup, then settle on 32MB — the current consensus after
Doom's "set it to 1GB" experiment failed.

#+begin_src emacs-lisp :tangle ./early-init.el
;;; early-init.el --- tangled from config.org -*- lexical-binding: t -*-
(setq gc-cons-threshold most-positive-fixnum)
(add-hook 'emacs-startup-hook
          (lambda ()
            (setq gc-cons-threshold (* 32 1024 1024))
            (message "Emacs ready in %s with %d garbage collections."
                     (emacs-init-time) gcs-done)))
#+end_src

Second: UI chrome. Disabling the toolbar /after/ the frame exists makes
the frame visibly flicker and resize. Setting frame parameters here means
the bars are never created at all.

#+begin_src emacs-lisp :tangle ./early-init.el
(push '(menu-bar-lines . 0) default-frame-alist)
(push '(tool-bar-lines . 0) default-frame-alist)
(push '(vertical-scroll-bars) default-frame-alist)
(setq frame-inhibit-implied-resize t
      inhibit-startup-screen t)
#+end_src

That startup-time message is real, by the way — check it:
[[elisp:(message "This session: started in %s" (emacs-init-time))][eval (emacs-init-time)]]

* Act II — Packages
:PROPERTIES:
:CUSTOM_ID: packages
:END:

Your nvim config bootstraps =lazy.nvim=. Emacs needs no equivalent
anymore: =package.el= (the installer) and =use-package= (the declarative
wrapper, like a lazy.nvim plugin spec) are both built in since Emacs 29.
The community drifted through =straight.el= and =elpaca= and largely came
back home.

#+begin_src emacs-lisp
;;; config.el --- tangled from config.org -*- lexical-binding: t -*-
(require 'package)
(add-to-list 'package-archives '("melpa" . "https://melpa.org/packages/") t)
(package-initialize)
(unless package-archive-contents     ; first run: fetch the package index
  (package-refresh-contents))
(require 'use-package-ensure)        ; load :ensure machinery eagerly --
(setq use-package-always-ensure t)   ; otherwise this setq is silently ignored
                                     ; until a form mentions :ensure explicitly
#+end_src

Anatomy of a ~use-package~ form, since you'll read thirty of them below:
~:init~ runs /before/ the package loads, ~:config~ after, ~:hook~ and
~:bind~ attach things *and* defer loading until needed — the same trick
as lazy.nvim's ~event=~ and ~keys=~.

* Act III — Your init.lua, translated
:PROPERTIES:
:CUSTOM_ID: defaults
:END:

Every option here is a line you already had in nvim. Same settings,
different names:

| init.lua                  | here                            |
|---------------------------+---------------------------------|
| ~number = true~           | ~display-line-numbers-mode~     |
| ~scrolloff = 10~          | ~scroll-margin~                 |
| ~ignorecase + smartcase~  | ~completion-ignore-case~ + evil |
| ~undofile = true~         | ~undo-fu-session~ (Act IV)      |
| ~splitright / splitbelow~ | ~evil-v/split-window-...~       |
| ~cursorline = true~       | ~global-hl-line-mode~           |
| ~timeoutlen = 300~        | ~which-key-idle-delay~          |
| ~clipboard = unnamedplus~ | default in GUI Emacs            |

#+begin_src emacs-lisp
(setq scroll-margin 8                  ; scrolloff
      scroll-conservatively 101        ; never recenter-jump while scrolling
      ring-bell-function #'ignore
      use-short-answers t              ; y/n instead of yes/no
      sentence-end-double-space nil
      completion-ignore-case t         ; the ignorecase family
      read-buffer-completion-ignore-case t
      read-file-name-completion-ignore-case t)

(add-hook 'prog-mode-hook #'display-line-numbers-mode)
(add-hook 'text-mode-hook #'display-line-numbers-mode)

(pixel-scroll-precision-mode 1)        ; smooth wheel scrolling
(electric-pair-mode 1)                 ; minimal autopairs
#+end_src

Things nvim does by default that Emacs makes you ask for — remembering
where you were, what you had open, what you typed in prompts:

#+begin_src emacs-lisp
(savehist-mode 1)                      ; minibuffer history survives restarts
(save-place-mode 1)                    ; reopen files at your last position
(recentf-mode 1)                       ; track recent files (SPC s .)
(global-auto-revert-mode 1)            ; pick up on-disk changes, like nvim
#+end_src

Emacs litters =file~= backups and =#file#= autosaves next to your work
unless told otherwise. Corral them, and keep =M-x customize='s machine
output out of this hand-written config:

#+begin_src emacs-lisp
(make-directory (locate-user-emacs-file "autosaves/") t)
(setq backup-directory-alist `((".*" . ,(locate-user-emacs-file "backups/")))
      auto-save-file-name-transforms `((".*" ,(locate-user-emacs-file "autosaves/") t))
      create-lockfiles nil
      custom-file (locate-user-emacs-file "custom.el"))
(load custom-file 'noerror 'nomessage)
#+end_src

* Act IV — Evil
:PROPERTIES:
:CUSTOM_ID: evil
:END:

The chapter that makes this place habitable. ~evil~ is not a vim
compatibility shim — it's a complete, arguably superior implementation
of vim's modal grammar on top of Emacs. Operators compose with text
objects and motions exactly as your fingers expect.

Every ~evil-want-*~ variable must be set *before* evil loads (they're
read once, at load time) — hence ~:init~, not ~:config~:

#+begin_src emacs-lisp
(use-package evil
  :init
  (setq evil-want-keybinding nil    ; evil-collection replaces evil's own extras
        evil-want-C-u-scroll t      ; C-u scrolls; prefix-arg moves to SPC u
        evil-want-C-i-jump nil      ; C-i is TAB in terminals; keep TAB working
        evil-want-Y-yank-to-eol t   ; Y = y$, fixing vim's oldest inconsistency
        evil-respect-visual-line-mode t   ; your j/k-through-wrapped-lines remap
        evil-undo-system 'undo-redo ; built-in since Emacs 28
        evil-search-module 'evil-search   ; vim-style / search with highlight
        evil-split-window-below t   ; splitbelow
        evil-vsplit-window-right t) ; splitright
  :config
  (evil-mode 1)
  ;; ESC in normal state also clears search highlight -- your <Esc> nohlsearch
  (advice-add 'evil-force-normal-state :after #'evil-ex-nohighlight)
  ;; ESC quits minibuffer prompts too, instead of stacking them
  (global-set-key [escape] #'keyboard-escape-quit))
#+end_src

Vanilla evil only covers editing buffers. ~evil-collection~ brings vim
keys to the /other/ two hundred places — dired, magit, help, package
list — so =j/k/gg/G= work everywhere you land:

#+begin_src emacs-lisp
(use-package evil-collection
  :after evil
  :config (evil-collection-init))
#+end_src

Your muscle memory, ported piece by piece. ~mini.surround~ →
~evil-surround~ (vim grammar: =ys=, =cs=, =ds=), and =gc= to comment —
nvim has it built in, evil doesn't:

#+begin_src emacs-lisp
(use-package evil-surround
  :after evil
  :config (global-evil-surround-mode 1))

(use-package evil-nerd-commenter
  :after evil
  :config
  (evil-define-key '(normal visual) 'global "gc" #'evilnc-comment-operator))
#+end_src

Your =TextYankPost= highlight autocmd, reimplemented as three lines of
advice on the yank operator (try =yap= on this paragraph after a restart):

#+begin_src emacs-lisp
(with-eval-after-load 'evil
  (require 'pulse)
  (advice-add 'evil-yank :around
              (lambda (orig beg end &rest args)
                (pulse-momentary-highlight-region beg end)
                (apply orig beg end args))))
#+end_src

Two upgrades nvim never gave you. *Persistent undo across sessions* (your
~undofile = true~) plus *vundo*, a navigable tree of every state this
buffer has ever been in — undo stops being a tightrope:

#+begin_src emacs-lisp
(use-package undo-fu-session
  :config (undo-fu-session-global-mode 1))

(use-package vundo
  :commands vundo)   ; bound to SPC t u below; h/l to walk, q to quit
#+end_src

And *avy*: type =gs=, then two characters you can see anywhere on screen,
and the cursor teleports there. After a week this replaces most counting
and =/= searching:

#+begin_src emacs-lisp
(use-package avy
  :after evil
  :config
  (evil-define-key '(normal visual) 'global "gs" #'avy-goto-char-2))
#+end_src

Finally, window navigation. In nvim you owned =C-h/j/k/l=. In Emacs,
=C-h= is the sacred help prefix — and we take it anyway, in normal state
only. Help stays on =F1= and =SPC h=, which read better from evil anyway:

#+begin_src emacs-lisp
(with-eval-after-load 'evil
  (dolist (state '(normal visual motion))
    (evil-define-key state 'global
      (kbd "C-h") #'evil-window-left
      (kbd "C-j") #'evil-window-down
      (kbd "C-k") #'evil-window-up
      (kbd "C-l") #'evil-window-right)))
#+end_src

Check your state machine from the inside — run this, then again from
insert mode (=i= first, then =C-c C-c= still works there):

#+begin_src emacs-lisp :tangle no
(message "evil state: %s | this buffer: %s" evil-state (buffer-name))
#+end_src

* Act V — The leader key
:PROPERTIES:
:CUSTOM_ID: leader
:END:

~which-key~ has been built into Emacs since 30.1 — the popup after a
pause is stock behavior now. Its delay is your nvim ~timeoutlen~:

#+begin_src emacs-lisp
(setq which-key-idle-delay 0.3)
(which-key-mode 1)
#+end_src

~general.el~ defines the =SPC= leader map. The layout below *is your nvim
layout*: =SPC s= search, =SPC w= windows, =SPC t= terminal/toggles,
=SPC b= buffers, =SPC TAB= alternate file, =SPC f= format, =SPC a= AI,
=SPC d= dotfiles. New citizens: =SPC SPC= M-x (Spacemacs-style), =SPC e=
eval (elisp evaluation), =SPC g= git, =SPC h= help, =SPC p= project,
=SPC q= quit.

Note the trick on =w=, =h=, =p=: we don't define those maps — we mount
Emacs' own prefix maps (=evil-window-map=, =help-map=,
=project-prefix-map=) onto the leader and inherit everything for free.
=SPC w v= and =SPC w s= split exactly like your nvim binds because
that's what evil's window map already contained.

#+begin_src emacs-lisp
(use-package general
  :after evil
  :config
  (general-create-definer my/leader
    :states '(normal visual)
    :keymaps 'override
    :prefix "SPC")

  (my/leader
    "SPC" '(execute-extended-command :wk "M-x")
    "TAB" '(evil-switch-to-windows-last-buffer :wk "alt buffer")
    "/"   '(consult-line :wk "search buffer")
    ":"   '(execute-extended-command :wk "M-x")
    "u"   '(universal-argument :wk "C-u prefix arg")
    "f"   '(apheleia-format-buffer :wk "format")
    "c"   '(async-shell-command :wk "shell command")

    "b"   '(:ignore t :wk "buffers")
    "bb"  '(consult-buffer :wk "switch buffer")

    "w"   '(:keymap evil-window-map :wk "windows")
    "h"   '(:keymap help-map :wk "help")
    "p"   '(:keymap project-prefix-map :wk "project")

    "s"   '(:ignore t :wk "search")
    "sf"  '(my/find-file-dwim :wk "files")
    "sg"  '(consult-ripgrep :wk "grep")
    "sw"  '(my/grep-word-at-point :wk "grep word")
    "s/"  '(consult-line-multi :wk "grep open buffers")
    "sh"  '(consult-info :wk "search the manuals")
    "sk"  '(embark-bindings :wk "keybindings")
    "sd"  '(consult-flymake :wk "diagnostics")
    "si"  '(consult-imenu :wk "symbols")
    "sr"  '(vertico-repeat :wk "resume last search")
    "s."  '(consult-recent-file :wk "recent files")

    "g"   '(:ignore t :wk "git")
    "gg"  '(magit-status :wk "magit")
    "gb"  '(magit-blame-addition :wk "blame")
    "gs"  '(diff-hl-stage-current-hunk :wk "stage hunk")
    "gr"  '(diff-hl-revert-hunk :wk "revert hunk")
    "gp"  '(diff-hl-show-hunk :wk "preview hunk")

    "t"   '(:ignore t :wk "terminal/toggle")
    "tt"  '(my/ghostel-toggle :wk "terminal toggle")
    "tT"  '(my/ghostel-below :wk "terminal (new below)")
    "tv"  '(my/ghostel-right :wk "terminal right")
    "tp"  '(ghostel-project :wk "terminal (project root)")
    "th"  '(eglot-inlay-hints-mode :wk "inlay hints")
    "tu"  '(vundo :wk "undo tree")

    "a"   '(:ignore t :wk "AI")
    "ac"  '(gptel :wk "chat buffer")
    "aa"  '(gptel-menu :wk "menu")
    "as"  '(gptel-send :wk "send region")
    "ar"  '(gptel-rewrite :wk "rewrite")

    "d"   '(:ignore t :wk "dotfiles")
    "df"  '(my/open-config :wk "this file")
    "dr"  '(my/reload-config :wk "reload config")
    "dd"  '((lambda () (interactive) (dired "~/.dotfiles")) :wk "dired dotfiles")

    "e"   '(:ignore t :wk "eval")
    "eb"  '(eval-buffer :wk "buffer")
    "ee"  '(eval-last-sexp :wk "last sexp")
    "ef"  '(eval-defun :wk "defun (top form)")
    "er"  '(eval-region :wk "region")
    "ep"  '(eval-print-last-sexp :wk "print last sexp")
    "e:"  '(eval-expression :wk "eval expression")
    "ei"  '(ielm :wk "ielm REPL")

    "q"   '(:ignore t :wk "quit")
    "qq"  '(save-buffers-kill-terminal :wk "quit")
    "qr"  '(restart-emacs :wk "restart")))
#+end_src

The three tiny commands referenced above — also your first real elisp.
~interactive~ is what turns a function into a command bindable to keys:

#+begin_src emacs-lisp
(defun my/open-config ()
  "Open this literate config."
  (interactive)
  (find-file (locate-user-emacs-file "config.org")))

(defun my/reload-config ()
  "Re-tangle config.org and load it -- apply edits without a restart."
  (interactive)
  (require 'ob-tangle)
  (org-babel-tangle-file (locate-user-emacs-file "config.org"))
  (load (locate-user-emacs-file "config.el") nil 'nomessage)
  (message "Dotfiles reloaded."))

(defun my/find-file-dwim ()
  "Project files when in a project, plain find-file otherwise."
  (interactive)
  (if (project-current)
      (project-find-file)
    (call-interactively #'find-file)))

(defun my/grep-word-at-point ()
  "Your <leader>sw: ripgrep the project for the word under point."
  (interactive)
  (consult-ripgrep nil (thing-at-point 'word t)))
#+end_src

Now press =SPC= and *wait a third of a second*. That popup is the map of
everything above. Explore it; nothing you launch can hurt you, and =C-g=
always backs out.

* Act VI — The minibuffer is your telescope
:PROPERTIES:
:CUSTOM_ID: completion
:END:

Telescope is one plugin; here the same power is four tiny ones sharing
the standard completion API. The win: *every* prompt in Emacs becomes a
fuzzy picker — not just the ones a plugin author blessed.

~vertico~ draws the vertical list. ~orderless~ lets you type space-separated
terms in any order (~init lua~ matches =init.lua=, =lua/init.el=...):

#+begin_src emacs-lisp
(use-package vertico
  :config
  (vertico-mode 1)
  (require 'vertico-repeat)            ; SPC s r -- your telescope resume
  (add-hook 'minibuffer-setup-hook #'vertico-repeat-save)
  ;; In file prompts, DEL eats a whole directory component
  (keymap-set vertico-map "DEL" #'vertico-directory-delete-char))

(use-package orderless
  :init
  (setq completion-styles '(orderless basic)
        completion-category-overrides '((file (styles basic partial-completion)))))
#+end_src

~marginalia~ annotates every candidate — docstrings beside commands, sizes
beside files. ~consult~ provides the actual pickers you bound under
=SPC s= (live-preview grep, line search, buffer switcher):

#+begin_src emacs-lisp
(use-package marginalia
  :config (marginalia-mode 1))

(use-package consult
  :init
  (setq xref-show-xrefs-function #'consult-xref       ; references in a picker
        xref-show-definitions-function #'consult-xref)
  (global-set-key [remap yank-pop] #'consult-yank-pop)) ; M-p after p: kill-ring picker
#+end_src

~embark~ has no nvim equivalent: it's a right-click menu for your
keyboard. Point at /anything/ — a file in a prompt, a URL in a buffer, a
candidate in a picker — press =C-.= and get every action that applies.

#+begin_src emacs-lisp
(use-package embark
  :bind (("C-." . embark-act))
  :init (setq prefix-help-command #'embark-prefix-help-command))

(use-package embark-consult
  :after (embark consult))
#+end_src

Exercises, in increasing order of enlightenment:

- [ ] =SPC SPC=, type a few letters out of order, watch orderless work.
- [ ] =SPC s g= and type slowly — results and *preview* update live.
  Then =C-.= on a result and choose ~export~: the live search becomes a
  regular editable buffer of matches.
- [ ] =SPC s h=, pick the /Emacs/ manual, search for "rectangle". You
  just full-text-searched the official docs without leaving home.

* Act VIII — Magit
:PROPERTIES:
:CUSTOM_ID: git
:END:

The single most common answer to "why do you still use Emacs?". Not a
git plugin — a full git UI that makes the index feel like a text buffer.
Stage by hunk or by *line* (visual-select inside a hunk, =s=), instant
fixups, interactive rebase as an editable buffer.

#+begin_src emacs-lisp
(use-package magit
  :commands (magit-status magit-blame-addition))
#+end_src

~diff-hl~ is your gitsigns: change markers in the fringe, hunk motions,
and the =SPC g s/r/p= hunk commands you already bound:

#+begin_src emacs-lisp
(use-package diff-hl
  :config
  (global-diff-hl-mode 1)
  (add-hook 'magit-pre-refresh-hook #'diff-hl-magit-pre-refresh)
  (add-hook 'magit-post-refresh-hook #'diff-hl-magit-post-refresh)
  (with-eval-after-load 'evil
    (evil-define-key 'normal 'global
      "]c" #'diff-hl-next-hunk
      "[c" #'diff-hl-previous-hunk)))
#+end_src

The lesson: =SPC g g= in any repo. Then press =?= — magit's entire
keymap unfolds as a menu. Stage with =s=, commit with =c c=, push with
=P p=. Five minutes in magit explains ten years of its reputation.

** Tree-sitter

Built in since Emacs 29; grammars compile on demand. ~M-x
my/treesit-install-all~ (checklist, chapter one) builds the set below,
then ~major-mode-remap-alist~ routes files into the ts-powered modes:

#+begin_src emacs-lisp
(setq treesit-language-source-alist
      '((bash "https://github.com/tree-sitter/tree-sitter-bash")
        (c "https://github.com/tree-sitter/tree-sitter-c")
        (cpp "https://github.com/tree-sitter/tree-sitter-cpp")
        (python "https://github.com/tree-sitter/tree-sitter-python")
        (rust "https://github.com/tree-sitter/tree-sitter-rust")
        (lua "https://github.com/tree-sitter-grammars/tree-sitter-lua")
        (json "https://github.com/tree-sitter/tree-sitter-json")
        (toml "https://github.com/tree-sitter-grammars/tree-sitter-toml")
        (yaml "https://github.com/tree-sitter-grammars/tree-sitter-yaml")
        (javascript "https://github.com/tree-sitter/tree-sitter-javascript")
        (typescript "https://github.com/tree-sitter/tree-sitter-typescript" nil "typescript/src")
        (tsx "https://github.com/tree-sitter/tree-sitter-typescript" nil "tsx/src")))

(defun my/treesit-install-all ()
  "Compile every grammar listed above (one-time, needs a C compiler)."
  (interactive)
  (dolist (lang (mapcar #'car treesit-language-source-alist))
    (unless (treesit-language-available-p lang)

* Act XII — Terminals
:PROPERTIES:
:CUSTOM_ID: terminals
:END:

~ghostel~ is a terminal emulator powered by *libghostty-vt* — the VT
engine from the Ghostty terminal, the same correctness and speed as a
standalone Ghostty but living inside an Emacs buffer. True color,
synchronized output, the Kitty keyboard and graphics protocols, inline
images, hyperlinks, desktop notifications, and shell integration
(directory tracking, prompt navigation) for bash/zsh/fish. No more
"shitty terminal" — this is the real thing. The native module is a
*prebuilt binary that auto-downloads on first use*: nothing to compile,
nothing to install from pacman.

#+begin_src emacs-lisp
(use-package ghostel
  :commands (ghostel ghostel-project ghostel-other)
  :init
  ;; Keys Emacs keeps for itself even in the terminal -- everything else
  ;; is forwarded to the program.  C-hjkl stay with Emacs so your window
  ;; motions work straight from the terminal.
  (setq ghostel-keymap-exceptions '("C-c" "C-x" "C-h" "C-j" "C-k" "C-l")))

;; Evil integration: ghostel-mode buffers get evil states like everything
;; else, with the terminal's own input modes underneath.
(use-package evil-ghostel
  :after (ghostel evil)
  :hook (ghostel-mode . evil-ghostel-mode))

(defun my/ghostel-below ()
  "Your <leader>tT -- a fresh terminal in a 12-line split below."
  (interactive)
  (select-window (split-window-below -12))
  (ghostel))

(defun my/ghostel-right ()
  "Your <leader>tv -- a fresh terminal in a 60-column split right."
  (interactive)
  (select-window (split-window-right -60))
  (ghostel))

(defun my/ghostel-toggle ()
  "Toggle a ghostel terminal in a 12-line split below.
If a ghostel window is showing, hide it.  Otherwise reuse the most
recent ghostel buffer (or spawn one) -- never stacks new terminals."
  (interactive)
  (let ((win (seq-find (lambda (w)
                         (eq (buffer-local-value 'major-mode (window-buffer w))
                             'ghostel-mode))
                       (window-list))))
    (if win
        (delete-window win)
      (let ((buf (seq-find (lambda (b)
                             (eq (buffer-local-value 'major-mode b) 'ghostel-mode))
                           (buffer-list))))
        (select-window (split-window-below -12))
        (if buf (switch-to-buffer buf) (ghostel))))))

;; C-hjkl are window motions even inside the terminal.  We keep them out
;; of the program via `ghostel-keymap-exceptions' above, then bind them to
;; evil's window commands in ghostel's default (semi-char) input map so
;; they actually move windows.  Guarded: the map name is only present once
;; ghostel has loaded.
(with-eval-after-load 'ghostel
  (when (boundp 'ghostel-semi-char-mode-map)
    (define-key ghostel-semi-char-mode-map (kbd "C-h") #'evil-window-left)
    (define-key ghostel-semi-char-mode-map (kbd "C-j") #'evil-window-down)
    (define-key ghostel-semi-char-mode-map (kbd "C-k") #'evil-window-up)
    (define-key ghostel-semi-char-mode-map (kbd "C-l") #'evil-window-right)))
#+end_src

Ghostel has several *input modes* that decide how much of your typing
the program sees: =C-c C-j= semi-char (the default — most keys go to the
shell), =C-c M-d= char (everything goes through; exit with =M-RET=),
=C-c C-l= line, =C-c C-t= copy mode (freeze the screen to select/yank
scrollback like any buffer). =SPC tt= toggles a drawer terminal below,
=SPC tp= opens one at the project root. (Worth one session someday:
~M-x eshell~, the shell written /in/ elisp where the prompt understands
Lisp. There is nothing like it outside Emacs.)

* Act XIV — Emacs 31 goodies
:PROPERTIES:
:CUSTOM_ID: emacs31
:END:

This Emacs is built from /master/. The 31 release branch has already been
cut, so master now calls itself *32.0.50* — but every goody the 31 cycle
introduced (the ones this chapter is named for) lives on master, plus
whatever 32 is growing. The chapters above still run on Emacs 30, so the
genuinely-new features gather here, each guarded with ~(when (>=
emacs-major-version 31) ...)~ — true on 31 and 32 alike.

That guard handles the /version/. It does not handle an option that was
renamed or dropped as master moved on (~native-comp-async-on-battery-power~
was one such — present in the 31 pretest, gone from 32.0.50). For that we
need a second, finer guard, below.

But first, your next rung of elisp. You've written a ~defun~ (Act V) and
bent an existing command with ~advice~ (Act IV); a /macro/ is the rung
above both. A function receives *values* and returns a value. A macro
receives your code *unevaluated* and returns new code, which Emacs then
runs in its place — you're writing a small program that writes programs.

~my/setopt-when-bound~ takes the same =VAR VAL VAR VAL= pairs as ~setopt~
and, for each, emits a ~setopt~ wrapped in a ~boundp~ test. The payoff is
real on master — an option renamed out from under you becomes a skipped
line, not a halted startup — but read the body as a recipe: backtick
=`= quotes a code template, =,var= drops a value into it, and =,@= splices
a /list/ of forms in. That is the whole of macro-writing in three commas.

#+begin_src emacs-lisp
(defmacro my/setopt-when-bound (&rest pairs)
  "`setopt' each VAR to VAL in PAIRS, skipping any VAR not bound here.
Keeps Emacs-31 options from breaking startup on a build (or an Emacs 30)
where a given option hasn't landed."
  (let (forms)
    (while pairs
      (let ((var (pop pairs)) (val (pop pairs)))
        (push `(when (boundp ',var) (setopt ,var ,val)) forms)))
    `(progn ,@(nreverse forms))))
#+end_src

See what it wrote: put point after the closing paren of any
~my/setopt-when-bound~ block below and =C-c C-v C-m= (~macroexpand~) — or
just =SPC h f= on ~setopt~ for what the underlying command does. =C-h i= →
/Elisp/ → "Macros" is the whole story when you want it.
